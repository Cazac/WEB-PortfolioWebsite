Strategic 3D Gameplay


During my time with Commanders of Valor, I have made tons of mechanics and prototypes, creating the grid-based combat experience that is used in the game today. The structure used is a complex system of action points that allows players to move units across the terrain, attack, use specials and switch between units freely with no set order. The team settled on using “A Star” pathfinding to move the units between the grid nodes on the map which generated the best overall experience for players or AI moving units. I also created an action queue to line up buffered actions chosen by the player to make the gameplay feel more consistent. Finally, the team used timed animation triggers to activate the methods used to refresh data, creating a more active feel to the hits of units.



System Architecture


Commanders of Valor was the first game I have had to create such a large scale set of cohesive systems that all needed to work together. The project went though some re-factoring on certain systems as new mechanics were added to fabricate a more consistent system architecture. However, each time I learned more techniques and better approaches to creating an improved design. Much of the game data had to be easily transported between scenes, save files and over the internet. This led to a strong appreciation of using Enums and wrapper classes to send and modify data in a legible way.



Creative Solutions


Commanders of Valour envisioned some creative mechanics that I was contracted to complete. One these mechanics that I had to pioneered was the family-tree system which used a branching pattern. This is not a new idea in any regard, but I could hardly come across any examples of how to do it during my research, so I had to design this by scratch. The math used for this system ends up becoming relatively complex rather quickly. For example, extra factors such as how many children per family, what the family relations are between each member and the spacing created from each of those nodes grows and changes exponentially. As a programmer who was still in school, there were many issues for me to tackle that I had little experience in. This ended up testing my problem-solving and research skills to the maximum, so that I could be more self sufficient. A lot of work did during my time with Commanders of Valor came down to pre-planning and document designing to understand exactly what was required of me as I created the envisioned game.



Online Multiplayer


Creating the online multiplayer for Commanders of Valour was a very fulfilling experience. Our ideal goal was to give the best connections to other players with the least amount of server connection requirements while at the same time, avoiding a reliance on other services. We ended up using Photon Unity Networking (PUN) to create basic match making to connect two clients together. Once that was achieved, one player was stated as the master client and the connection is directly setup between the two players. Players then communicated to each other though Remote Procedure Call (RPC) codes to convey each players actions directly to one another. This achieved our goals of having a very minimal connection to other services while utilizing the best possible online transmission methods for our style of turn-based strategy game.

Crossing The Cosmos

Crossing the Cosmos is a hardcore high stakes resource management game. Fly across space while managing the SS PROMETHEUS to keep your resources high and your crew alive. It will be a long and deadly voyage to your new home in “New Terra”. So, make sure you are ready, Captain!

Key Features

Crossing the Cosmos is a hardcore high stakes resource management game that was originally intended to be a weeklong game jam. However, the game was finished as a part of the "Finish Your Game" Game Jam as we became so passionate about the project that we ended up creating additional mechanics, to produce a more polished game experience. The final project left the team feeling quite happy with what we made together. Personally, I found this project super interesting and helpful to my growth as a game developer. I learned more about game design aspects to develop the game from scratch as well working with more junior project members and learning how best to mentor them.


My major roles and tasks done on this project include:

- Developed game design outlines and documents
- Programmed and designed system architecture
- Created and implemented UI/UX design
- Envisioned and programed key randomization systems
- Play tested and balanced difficulty scaling
- Wrote creative writing scripts and story

2D Gameplay


Crossing the Cosmos uses many different 2D gameplay systems that I have since become very acquainted with, such as AI pathing for the moving characters on screen, both frontend and backend systems for the UI interactions, and polishing animations for both the UI and the main ship to breath life into the game. Many of the backend systems required complex randomization systems that take the players current state into account, such as selecting chained quests that appear more frequently with each passing day if conditions are met.



Unity Scriptables


Crossing the Cosmos uses a complex system for handling Unity’s scriptable objects which I created myself. The front end of the system uses a very user-friendly approach to make it easier for non-programmers to input new data. The goal was to streamline the storyline events with complex options and outcomes to an easier and manageable system for all developers no matter their role. The design allows for users to work with a simpler workflow of a Word documents or Excel sheets. If you want to read more about this system and the Unity Scriptable Exporter project, then click here.



Design & Balance


As Crossing the Cosmos had more linear player options with simpler mechanics and focused on turn-based resource management, designing the game and balancing the ecosystem was on a smaller scale then other games I had previously worked on. However, this project was far more intensive and sensitive to subtle changes. The work I did for this game helped me familiarize myself with processes and designs used to balance games on a more effective scale through spreadsheets as well as tweaking numbers and mechanics to create the most positive experience for the player.

Do you love Unity’s Scriptable Objects system? Do you hate the lack of spellcheck and inability to see all of your data side by side? Do you also find it annoying to copy and paste everything directly from a word document to be used in your scriptable objects?

Introducing my newest Unity application the Unity Scriptable Exporter! Convert vast amounts of data to scriptable objects in an easy-to-use format made for even the non-programmer!

Key Features

This application allows you keep all of your raw data information and perform spellchecking outside of Unity’s more clunky systems. This allows for a great workflow from Excel into Scriptable Objects all in one easy application.

Content Workflow


To use the Unity Scriptable Exporter, you have to is to add a sample of the scriptable you want to work with, generate an Excel template of the scriptable object and fill out the data you need to add into the scriptable Excel template. After that, you need to load the Excel file and the sample scriptable into the application for which it will generate scriptable objects based off the Excel data fed to it. Finally, you will need a GUID taken from a metafile of the sample scriptable from inside of Unity so that you can export your files. Unity requires the GUID in order for the application can access the metafiles Unity uses. By following these steps, you will be able to use the Unity Scriptable Exporter to move your data into scriptable files with ease.



Programming & Design


The development of this application spanned multiple different technologies in my quest to learn how to program alongside Excel file formats or more commonly know as CSV files. I also needed to learn how to work with Unity files so that they were mergeable through the application. The GUID requirement was a unique solution to the roadblock of processing Unity files as Unity requires a GUID to match the scriptable object script to the scriptable when loaded into the system. Developing this tool required me to put together a lot of the different skills that I had learned in data processing as well as UI design. Each of the individual variables needed to be matched to unexpected template leaving a lot of catch cases to filter out bad data while still outputting useable data.

Image Based Stenography

Cryptography has always been a very fascinating subject to me. So, as a final project for my cryptography course at KPU, I decided it would be very interesting to hide messages inside of images. Essentially, these messages could be sent between two people without interception of a third-party. When viewed directly the result is an image, but then this image could be transformed back into a message when opened within the application. So, if you ever wanted to send a secret message, here’s how!

Encryption & Decryption Design


To create the encryption application, I took an image file of either PNG or JPEG and imported it into the application. From there it was decoded the photo file into its hexadecimal format. The hexadecimal format represented the colours that composed the image which was then modifiable. Once the application was run with an input message, I would modify the values from the top left downwards to inscribe a hidden code within the colours of the image. To the naked eye, this would appear as a little discolouration or artifacts from the image, but when it was run back into the decryption side of the software, the message would reappear as it’s read back from key points in the hexadecimal image color scheme. An alternative way to store the information could have been directly inside the file’s code with no visual artifacting, but unfortunately this was not within the scope of the project that I created.
